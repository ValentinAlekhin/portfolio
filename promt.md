# 3. Nuxt UI

Используй Nuxt UI как основную UI-библиотеку.

Подключи:

```ts
export default defineNuxtConfig({
  modules: [
    '@nuxt/ui'
  ]
})
```

Используй официальный способ подключения CSS для установленной версии Nuxt UI.

Основной CSS-файл должен содержать интеграцию Tailwind CSS и Nuxt UI, например:

```css
@import "tailwindcss";
@import "@nuxt/ui";
```

Не подключай PrimeVue и `@primevue/nuxt-module`.

Не добавляй `@nuxtjs/color-mode` в массив `modules`, если установленная версия Nuxt UI регистрирует его автоматически.

При этом настрой color mode через поддерживаемые Nuxt UI параметры в `nuxt.config.ts`.

Nuxt UI используй для элементов, где библиотека даёт полезное поведение, консистентность и доступность:

* кнопки;
* icon buttons;
* badge;
* tooltip;
* dropdown menu;
* мобильное меню;
* drawer или slideover;
* переключатель темы;
* переключатель языка;
* доступные интерактивные элементы.

Для переключения темы используй подходящий компонент Nuxt UI:

* `UColorModeButton`;
* `UColorModeSelect`;
* либо собственный компонент на основе API Nuxt UI.

Выбирай только те компоненты и API, которые действительно существуют в установленной стабильной версии Nuxt UI.

Не копируй API компонентов из памяти без проверки типов установленного пакета.

Не оборачивай каждый HTML-элемент в компонент Nuxt UI.

Для основной структуры используй семантический HTML:

* `header`;
* `nav`;
* `main`;
* `section`;
* `article`;
* `footer`;
* `button`;
* `a`;
* `h1`–`h3`;
* `ul`;
* `ol`.

Настрой собственную визуальную систему.

Сайт не должен выглядеть как стандартный Nuxt UI starter или готовый шаблон.

Используй:

* настройки Nuxt UI в `app.config.ts`;
* семантические цвета;
* CSS-переменные;
* Tailwind CSS;
* `ui` props компонентов Nuxt UI, когда это оправдано;
* локальные UI-компоненты для повторяющихся паттернов.

Не создавай `BaseButton.vue`, если достаточно стандартного `UButton`.

Собственные обёртки создавай только при наличии повторяющейся проектной логики или сложного общего оформления.

Корневое приложение оформи в соответствии с требованиями Nuxt UI. Если текущая версия библиотеки требует `UApp`, используй его в `app.vue`.

# 4. Формат приложения

Сайт должен быть:

* одностраничным по смыслу;
* мультиязычным;
* предварительно отрендеренным;
* статически генерируемым;
* без backend;
* без серверной базы данных;
* без обязательного Node.js-сервера после публикации;
* пригодным для любого обычного статического хостинга.

Не отключай SSR через:

```ts
ssr: false
```

Сайт должен генерировать полноценный HTML для каждой локали, а не быть SPA с пустым HTML-контейнером.

Основная команда генерации:

```bash
pnpm generate
```

Готовые файлы должны появляться в стандартном каталоге статического вывода текущей версии Nuxt, обычно:

```text
.output/public
```

Именно содержимое каталога статического вывода должно быть пригодно для загрузки на хостинг.

Добавь в `package.json` как минимум следующие команды:

```json
{
  "scripts": {
    "dev": "nuxt dev",
    "build": "nuxt build",
    "generate": "nuxt generate",
    "preview": "nuxt preview",
    "lint": "eslint .",
    "lint:fix": "eslint . --fix",
    "typecheck": "nuxt typecheck"
  }
}
```

Настрой предварительную генерацию:

* `/`;
* `/ru`.

Проверь, что обе страницы присутствуют в статическом выводе после `pnpm generate`.

При необходимости явно добавь маршруты в настройки prerender, если автоматический crawler не гарантирует генерацию обеих локалей.

Не должно быть runtime-зависимости от Nuxt Content server API после публикации.

# 7. Хранение контента

Весь пользовательский и маркетинговый контент страницы должен храниться в YAML через Nuxt Content.

Не размещай основные содержательные тексты непосредственно в Vue-компонентах.

Допустимы только технические строки:

* fallback-сообщения;
* aria-атрибуты;
* названия внутренних состояний;
* сообщения об ошибке;
* короткие служебные подписи.

Используй структуру:

```text
content/
  portfolio/
    en.yml
    ru.yml
```

Создай:

```text
content.config.ts
```

Используй:

* `defineContentConfig`;
* `defineCollection`;
* коллекцию типа `data`;
* Zod-схему;
* `queryCollection`;
* автоматически генерируемые типы Nuxt Content.

Не создавай собственную ручную копию типа, которая полностью повторяет схему и может рассинхронизироваться с ней.

Отдельные вспомогательные TypeScript-типы допустимы для props, UI-состояний и производных данных.

Пример структуры коллекции:

```ts
import { defineCollection, defineContentConfig, z } from '@nuxt/content'

export default defineContentConfig({
  collections: {
    portfolio: defineCollection({
      type: 'data',
      source: 'portfolio/*.yml',
      schema: z.object({
        locale: z.enum(['en', 'ru'])
      }).passthrough()
    })
  }
})
```

Не оставляй схему с одним `locale` и `passthrough()` в финальной реализации.

Опиши полноценную строгую схему всех полей.

Схема должна:

* проверять обязательные поля;
* разрешать действительно необязательные поля;
* проверять URL там, где URL обязателен;
* поддерживать пустые или отсутствующие контакты;
* поддерживать неопубликованные проекты;
* поддерживать отсутствие фотографии;
* поддерживать отсутствие метрик;
* поддерживать отсутствие location и timezone;
* генерировать корректные TypeScript-типы.

# 21. Анимации

Анимации должны быть спокойными и функциональными.

Используй:

* Vue `<Transition>`;
* VueUse;

Допустимые эффекты:

* плавное появление секций;
* небольшое вертикальное смещение;
* лёгкое движение карточек при наведении;
* мягкое увеличение изображения проекта;
* изменение header после прокрутки;
* небольшая анимация точки доступности;
* плавная смена визуальных состояний;
* деликатное движение декоративного gradient.

* тяжёлый parallax;
* scroll hijacking;
* горизонтальный принудительный scroll;
* WebGL;
* Three.js;
* 3D-сцены;
* длинный preloader;
* анимацию каждой буквы;
* анимацию каждого слова;

```text
prefers-reduced-motion
```

* отключай появления со смещением;
* отключай декоративные бесконечные анимации;
* отключай smooth scroll;
* убирай scale-анимации, способные вызывать дискомфорт;
* показывай контент сразу.

Контент не должен оставаться невидимым, если JavaScript отключён или IntersectionObserver не сработал.
